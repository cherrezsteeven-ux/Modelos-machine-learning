Metodología y justificación del modelo

Variable objetivo y predictores

Se predice macro.area (Norte-Centro, Cerdeña, Sur) a partir de los 8 ácidos grasos (variables numéricas). La variable region se excluyó deliberadamente del modelo: es una clave que determina macro.area de forma perfecta y unívoca (cada región pertenece a una única macro-área), por lo que incluirla produciría una fuga de información (data leakage) y un modelo trivial que no aprende nada sobre la química del aceite. En este prototipo, region se usa solo como acceso rápido para cargar un perfil típico de composición y explorar el modelo — no entra como insumo de la predicción.

Partición train/test

Se usó un split estratificado 80% / 20% (457 muestras de entrenamiento, 115 de prueba), manteniendo la proporción de clases en ambos conjuntos. Con solo 572 observaciones y una clase minoritaria de 98 casos (Cerdeña), una proporción como 50/50 o 70/30 dejaría muy pocos ejemplos por clase para que los árboles aprendan reglas robustas; 80/20 preserva suficiente señal de entrenamiento por clase (≈78 casos de Cerdeña en train) sin sacrificar un tamaño de prueba razonable (≈115 casos, ~20 por clase minoritaria) para estimar el desempeño con relativa estabilidad. Se complementó con validación cruzada estratificada (5 folds) sobre el conjunto de entrenamiento para confirmar que el resultado no depende de una sola partición.

Los tres modelos separan perfectamente las tres macro-áreas: es un resultado genuino y ya documentado en la literatura de este conjunto de datos clásico (Forina et al.), no una fuga de datos — el perfil de ácidos grasos (sobre todo oleico, eicosenoico y linoleico) difiere marcadamente entre el sur, el centro-norte y Cerdeña por razones agroclimáticas. Se eligió Random Forest como modelo final para el prototipo por ser más robusto que un único árbol (menor varianza, menos sensible a una muestra atípica) y más simple de explicar de forma estable con SHAP que Gradient Boosting, cuya salida por etapas es más sensible a la elección de hiperparámetros.

Métrica principal: F1-macro

Se eligió F1-macro como métrica principal en lugar de accuracy porque las clases están desbalanceadas (Sur 323, Norte-Centro 151, Cerdeña 98 muestras): con accuracy simple, un modelo que acertara siempre "Sur" ya obtendría ~56% sin haber aprendido nada útil sobre Cerdeña o Norte-Centro. F1-macro promedia el F1 de cada clase con el mismo peso, así que penaliza con la misma fuerza los errores en la clase minoritaria (Cerdeña) que en la mayoritaria (Sur) — algo relevante para un cliente que necesita identificar correctamente todas las macro-áreas de origen, no solo la más común, por ejemplo para fines de trazabilidad o control de fraude de denominación de origen.

Explicabilidad (SHAP)

El gráfico de la derecha muestra valores tipo Shapley calculados con un algoritmo de aproximación por muestreo de permutaciones (equivalente al enfoque interventional/Kernel SHAP): se generan permutaciones aleatorias del orden de las variables y, partiendo de un conjunto de referencia (45 muestras reales tomadas del set de entrenamiento, estratificadas por clase), se mide cuánto cambia la probabilidad predicha al ir "revelando" cada variable — el promedio de esos cambios a través de muchas permutaciones es la contribución (SHAP) de esa variable. Este cálculo corre íntegramente en el navegador, junto con los 60 árboles del Random Forest, ambos embebidos en este archivo HTML.
